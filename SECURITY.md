# Security policy

## Reporting a vulnerability

**Do not** open a public GitHub Issue or Discussion for security vulnerabilities.

Email **security@nt2.me** with:

- A clear description of the issue
- Steps to reproduce (if applicable)
- Impact assessment if known

Please allow reasonable time for assessment and fix before any public disclosure.

## Out of scope for this repository

Public Issues here are for **sanitised** product bugs and feature requests only. See [README.md](./README.md) for channel routing (billing → `support@nt2.me`, privacy → `privacy@nt2.me`).
